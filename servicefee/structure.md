# Структура проекта «Калькулятор сборов АВИА»

## Корень проекта
- `CURRENT_STAGE.md` — описание текущего этапа и следующих шагов.
- `structure.md` — этот файл, структура файлов и каталогов.
- `CHANGELOG_AI.md` — журнал изменений, вносимых ИИ.

## avia-calc/

```
avia-calc/
├── api.php                   # Единая точка входа для AJAX (get_initial_data, calculate, get_history, clear_history, delete_history_item)
├── admin.php                 # Админка правил: список клиентов, секции правил (серв.сбор, трансгран, эквайринг, спецправила)
├── index.php                 # Страница калькулятора: вкладки Калькулятор / История, форма ввода, блок результата
├── assets/
│   ├── admin.js              # Логика админки: клиенты, правила (серв.сбор, трансгран, эквайринг, спецправила)
│   ├── app.js                # Логика интерфейса: App.init, расчёт, история, копирование
│   └── style.css             # Стили: калькулятор + админка (списки, таблицы, модальное окно)
├── config/
│   └── database.php          # Настройки подключения к БД (путь к SQLite, schema, seed)
├── database/
│   ├── schema.sql            # SQL-схема создания таблиц
│   ├── seed.sql              # Начальные данные (клиенты, правила и т.д.)
│   └── avia_calc.sqlite      # Файл БД (создаётся автоматически при инициализации)
└── src/
    ├── Calculator.php        # Класс App\Calculator — полный расчёт сборов (сервисный + трансгран + эквайринг), формулы, мультивалютность
    ├── CurrencyConverter.php # Класс App\CurrencyConverter — конвертация валют, курсы РС ТЛС / Казахстана, мультивалютный результат
    ├── AdminManager.php      # Класс App\AdminManager — CRUD правил (клиенты, серв.сбор, трансгран, эквайринг, спецправила)
    ├── Database.php          # Класс App\Database — синглтон-обёртка над PDO для SQLite
    ├── HistoryManager.php    # Класс App\HistoryManager — сохранение, чтение, удаление записей истории расчётов, cleanup по лимиту
    └── RuleEngine.php        # Класс App\RuleEngine — загрузка и кэширование правил из БД, методы получения правил
```

### Таблицы БД (schema.sql)
- `clients` — категории клиентов (РТ, РМЕ, SDG и т.д.).
- `service_types` — типы услуг (выписка, EMD, возврат, обмен).
- `payment_methods` — способы оплаты (счёт, карта БРС, карта КЗ и т.д.).
- `service_fee_rules` — правила сервисного сбора (клиент + тип услуги).
- `transgran_rules` — правила трансграничного сбора (клиент, тип услуги, колонка).
- `transgran_exceptions` — исключения трансграна (например, поставщик KZ).
- `acquiring_rules` — правила эквайринга (клиент + способ оплаты).
- `client_special_rules` — спецправила клиента (например, не брать стандартный трансгран).
- `calculation_history` — история расчётов (JSON вход/результат, формулы).
